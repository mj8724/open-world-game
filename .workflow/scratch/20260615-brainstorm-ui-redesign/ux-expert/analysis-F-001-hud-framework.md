# Analysis: F-001 HUD基础框架

## 1. Feature Context & UX Objectives
HUD 基础框架是玩家与开放世界交互的常驻视窗。从 UX 视角来看，其核心目标是提供关键信息的**常驻可见性**与**无干扰性**。玩家需要随时掌握经济命脉与发展进度，但又不能被冗余的 UI 元素剥夺对游戏主画面的注意力。

## 2. Decision Impacts & Constraints
- 遵循 **D-UI-02** 与 **D-UI-03**：HUD 采用拟玻璃化风格及底层深色半透明遮罩与动态模糊。UX 层面上，这 MUST 确保在任何游戏地形（如雪地、强光照射区）下，资源数字和图标的对比度依然达标。
- 遵循 **D-PM-02**：经济产出和消耗 MUST 做到“所见即所得”。
- 遵循 **D-SME-01**：金钱、木材/矿物、电力、人口及科技进度 MUST 永久常驻。

## 3. UX & Interaction Strategy
### 3.1 信息架构 (Information Architecture)
- **顶部/角落分组**：建议将静态或全局性的资源（人口、科技进度）与高频变动的资源（金钱、电力）进行视觉分组。
- **层级区分**：当前总量 MUST 作为第一视觉层级（最大字号）；速率变化（如 `+15/s`）MUST 作为第二层级，紧随其后。

### 3.2 交互模式 (Interaction Patterns)
- **Hover 探查 (Hover for Details)**：常驻信息保持极简，当玩家将鼠标悬停于资源项时，MUST 弹出含有详情（如产出明细、消耗来源统计）的轻量级 Tooltip。
- **动态数字反馈 (Dynamic Counting)**：资源变动时，数值的增减 MUST 伴随正向（绿色）或负向（红色）的短暂飘字或颜色渐变反馈，以满足 D-PM-02 的直观感知需求。

### 3.3 可用性与无障碍 (Usability & Accessibility)
- **视觉锚点**：图标 MUST 具备强识别度的轮廓，避免纯粹依赖颜色区分资源类型（照顾色弱玩家）。
- **认知负荷**：限制屏幕常驻元素的绝对数量。科技树进度可简化为环形进度条或简明图标，避免长文本常驻。
