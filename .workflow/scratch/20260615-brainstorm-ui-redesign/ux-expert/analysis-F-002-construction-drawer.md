# Analysis: F-002 边缘滑出式建设面板

## 1. Feature Context & UX Objectives
建设系统是游戏经济与扩张的核心循环之一。传统的全屏或中心弹出式面板会打断玩家对局势的实时监控。因此，边缘抽屉面板（Drawer Panel）的引入是为了保持游戏流程的连贯性，实现“边观察边决策”的 UX 目标。

## 2. Decision Impacts & Constraints
- 遵循 **D-UX-01**：建设等二级菜单 MUST 使用从边缘滑出的抽屉式面板，绝对不遮挡主视口中心。
- 遵循 **D-PM-01**：此项为 P0 级优先处理的核心基础循环。

## 3. UX & Interaction Strategy
### 3.1 抽屉面板交互 (Drawer Interaction)
- **呼出机制**：面板 MUST 支持通过快捷键（如 `B`）与屏幕边缘的显性唤出按钮（Affordance）双重触发。
- **过渡动效**：滑出动效 MUST 平滑自然，遵循物理惯性（如 Ease-Out 曲线），时长控制在适中范围，避免拖沓感。
- **空间分配**：抽屉面板的宽度 SHOULD 限制在屏幕宽度的 20%-25% 之间，既能容纳两到三列的建筑缩略图，又留出足够的视口观察环境。

### 3.2 信息架构与分类 (Taxonomy)
- **快速过滤**：建筑分类（经济、军事、防御等）MUST 使用直观的页签（Tabs）或垂直图标导航栏进行快速切换。
- **卡片设计**：每个建筑选项卡片 MUST 包含缩略图、核心造价以及状态指示（可建/资源不足/未解锁）。

### 3.3 错误预防与状态管理 (Error Prevention & State)
- **资源短缺提示**：当资源不足时，建筑卡片 MUST 置灰，并使用红色高亮显示缺少的具体资源项。
- **前置条件限制**：如果缺少前置科技，MUST 在 Hover 时明确告知玩家所需解锁的条件，而不是提供无意义的死胡同。
- **拖拽与点击放置**：支持单击选中后在主视口预览模型，或直接从抽屉拖拽至地形的双重交互模式，以适应不同玩家的心智模型。
