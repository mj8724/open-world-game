# Analysis: F-003 单位详情与指令面板

## 1. Feature Context & UX Objectives
单位是游戏互动的主体，其详情与指令面板必须在信息密度（战斗属性）与操作效率（发布指令）之间取得平衡。复杂的嵌套操作逻辑需要被扁平化，使得玩家能在激烈的即时战斗中迅速读取关键数据并做出反应。

## 2. Decision Impacts & Constraints
- 遵循 **D-UX-02**：选中单位后，其详细状态与指令菜单 MUST 固定于屏幕指定区域（如右下角），建立稳定的肌肉记忆。
- 遵循 **D-SME-02**：单位的“掩体加成”与“护甲抗性”等关键战斗收益数据 MUST 获得强化显示。

## 3. UX & Interaction Strategy
### 3.1 布局与信息层次 (Layout & Hierarchy)
- **区域划分**：右下角面板 MUST 分为三个清晰的区域：
  1. **身份与核心状态区**：头像/名称、血量/护盾条。
  2. **战斗属性区**：强调显示 **D-SME-02** 要求的掩体加成和抗性图标。
  3. **指令动作区**：扁平化排列的技能或命令按钮。
- **掩体与抗性可视化**：与其使用枯燥的数字，不如使用“盾牌”、“防弹衣”等具有隐喻的图标，并结合颜色（如绿色表示受到掩体保护）来传达状态。

### 3.2 交互指令 (Command Interactions)
- **指令响应**：技能和指令按钮 MUST 有明确的冷却（Cooldown）态与可用态。
- **快捷键映射**：面板上的每个核心指令 MUST 在 UI 上标注对应的键盘快捷键，降低对鼠标点击的依赖。
- **群组选择适配**：当选中多个单位时，面板 MUST 智能折叠个体差异信息，转而显示队伍的整体构成比例和共同的可用指令集合。

### 3.3 状态管理与微交互 (State & Microinteractions)
- **焦点切换**：从空状态（未选中）切换到选中状态时，面板 MUST 有一个轻快的弹入（Pop-in）或滑入动画，确认焦点已转移。
- **伤害反馈**：当选中单位正在受击时，面板内的血量条 MUST 具备强烈的视觉闪烁或震动反馈，提醒玩家进行微操救援。
